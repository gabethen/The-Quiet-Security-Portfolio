# Mission 03: Network Traffic Analysis

## Objective
Learn how security analysts examine network activity to identify suspicious behavior.

## Scenario
A company notices unusual network communication from one workstation. Your task is to analyze traffic patterns and determine if there is a possible threat.

## Analyst Questions
- Who is communicating?
- What systems are involved?
- What ports and protocols are being used?
- Is the traffic expected?
- Does the behavior match known threats?

## Evidence Sources
- Packet captures (PCAP)
- Firewall logs
- DNS queries
- IP addresses
- Network monitoring tools
- IDS alerts

## Skills Practiced
- Understanding network communication
- Identifying abnormal traffic
- Investigating indicators of compromise (IOCs)
- Basic threat hunting

## Outcome
By the end of this mission, you will understand how network data helps analysts detect and investigate security incidents.
